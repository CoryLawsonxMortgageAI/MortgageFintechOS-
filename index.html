MortgageFintechOS — Technology Stack Referencev3.0 Production

This system is a production-grade autonomous AI operating system for mortgage lending. It uses 13 specialized AI agents orchestrated by a central daemon, with 9 external service integrations, real-time task dispatch, and regulatory compliance built-in. Below is a complete reference of every technology, architecture pattern, and key concept.

Core ArchitecturePython 3.12+
Orchestrator Pattern (Daemon)
Central long-running process managing agent lifecycle, task dispatch, scheduling, health monitoring, and state persistence. Uses asyncio event loop for non-blocking concurrent operations.

Task Queue — Priority-based FIFO queue with 4 levels: CRITICAL, HIGH, MEDIUM, LOW. Supports retry with exponential backoff (2^n seconds). History tracking for audit trail.

Watchdog Supervision — Crash loop detection for subsystems (dispatch, scheduler, health_monitor). Auto-restart with 30s polling. Creates GitHub issues after 5+ crashes in 5 minutes.

State Persistence — Debounced writes to disk via StateStore. Agent state, task queue, and Paperclip budgets survive restarts.

Scheduled Jobs — DailyScheduler with time-of-day triggers, interval support, and day-of-week filtering. 11 scheduled jobs from 02:00 to 10:00 daily.
AI Agent System — 13 Agents / 4 Divisions
Mortgage Ops Division (4 Agents)
DIEGO — Pipeline Orchestration: Loan triage (FHA/VA/CONV/USDA/JUMBO), stage progression, priority assignment, bottleneck detection
MARTIN — Document Intelligence: Classification (W2/paystub/bank/tax), OCR validation, fraud detection (font/layers/dates), GHOST OSINT borrower verification
NOVA — Income & DTI: W-2 dual-method (FHA HB 4000.1), Schedule C self-employment, DTI ratio with compensating factors, collections 5% rule
JARVIS — Condition Resolution: LOE templates, condition-to-doc mapping, compliance citations (FHA/FNMA/FHLMC), condition lifecycle

Codebase Engineering (6 Agents)
ATLAS — Full-Stack: LLM-powered code generation, API scaffolding, GitHub branch/commit/PR creation
CIPHER — Security: OWASP scanning via GitHub alerts, SOC2/PCI/GLBA compliance, encryption audit, PentAGI integration
FORGE — DevOps: GitHub Actions workflow triggers, deployment dispatch, CI/CD pipeline YAML generation
NEXUS — Code Quality: PR review (real diff + LLM analysis), test generation (pytest), tech debt analysis, SOLID refactoring
STORM — Data Engineering: ETL pipeline generation, HMDA compliance reporting, ULDD export specs
SENTINEL — Intelligence: Codebase scanning, trend analysis, reverse engineering, AutoResearch (Karpathy-style), deep security audit

Growth Ops Division (3 Agents)
HUNTER — Dev Lead Discovery: GitHub/HN/Reddit scanning, keyword-weighted scoring, tier classification (hot/warm/cold)
HERALD — Build-in-Public: LLM content generation, content calendar, multi-platform (Twitter/dev.to/LinkedIn)
AMBASSADOR — Community Engagement: Rate-limited GitHub starring/commenting, anti-spam guardrails, relationship tracking
External Integrations (9 Services)
GitHub API v3 — Full code operations: repository CRUD, branch management, pull requests (create/review/merge), GitHub Actions (trigger/monitor/cancel workflows), security scanning (code scanning/Dependabot/secret scanning alerts). Bearer token auth.

Notion API v2025-09-03 — Knowledge base + audit trail. data_source_id parent (new v2025 concept replacing database_id). Page CRUD, block management, search, agent result auto-sync.

Google Drive API v3 — Document source for borrower docs. Service account JWT auth (google-auth). File listing, download, Google Docs export, batch folder import → MARTIN classification.

Wispr Flow — Voice-to-text via webhook. HMAC-SHA256 signature validation. Keyword-based agent routing with confidence scoring. Auto-Notion sync.

LLM Router (Multi-Model) — Intelligent routing to OpenAI, Anthropic (Claude Sonnet 4.6), OpenRouter. Task-based routing matrix: code_gen→Claude, data_eng→GPT-4o, security→Claude. Agent-specific overrides. Automatic fallback.

Paperclip AI — Enterprise orchestration with Board approval governance. Per-agent budgets ($300-800/mo), auto-pause at 100%, immutable audit log. Ticket lifecycle: open→approved→in_progress→completed.

GHOST OSINT CRM — Entity investigation for fraud detection. Entity creation/search/network graphs, relationship mapping, OSINT lookups (email/phone/domain/person), risk scoring, borrower verification.

PentAGI — Autonomous penetration testing. 111+ security tools. Scan creation, vulnerability discovery, attack surface analysis, report generation, autonomous task submission (recon/exploit/report).

Browser Automation — Headless HTTP client for Growth Ops. Token-bucket rate limiter per domain, blocked domain guardrails, action budgets per platform/hour, persistent session store, fingerprint rotation, auto-429 backoff.
Key Technical Terms & Concepts
Agentic AI — AI systems that autonomously plan, execute, and adapt. Each agent has specialized domain knowledge and can invoke tools (GitHub, Notion, LLM) without human intervention.

Multi-Agent Orchestration — Central daemon coordinates 13 agents via task queue. Agents share integrations but operate independently. Cross-agent pipelines: DIEGO→MARTIN→NOVA→JARVIS (mortgage), ATLAS→NEXUS→CIPHER→FORGE (engineering).

LLM Routing Matrix — Maps task categories to optimal models: code_generation→Claude, security_analysis→Claude, data_engineering→GPT-4o, document_processing→Claude. Includes agent-level overrides and automatic fallback chains.

RAG (Retrieval-Augmented Generation) — Pattern where agents retrieve context (documents, code, Notion pages) before LLM generation. Used by MARTIN (doc classification), NEXUS (PR review), SENTINEL (codebase analysis).

DAG (Directed Acyclic Graph) — Task dependency graph for pipeline execution. DIEGO manages loan stage progression. STORM builds ETL DAGs for data pipelines.

OWASP Top 10 — Standard security vulnerability categories. CIPHER maps GitHub code scanning alerts to OWASP categories (injection, XSS, CSRF, etc.) for compliance reporting.

FHA HB 4000.1 — FHA Single Family Housing Policy Handbook. NOVA uses Section II.A.5.b for W-2 income, II.A.4.d.v for collections 5% rule. JARVIS references for condition mapping.

HMDA / ULDD — Home Mortgage Disclosure Act (regulatory reporting) and Uniform Loan Delivery Dataset (Fannie Mae/Freddie Mac delivery format). STORM generates compliant exports.

DTI (Debt-to-Income) — Ratio of monthly debt payments to gross monthly income. NOVA calculates front-end (housing) and back-end (total) DTI with compensating factors per FHA/FNMA guidelines.

LOE (Letter of Explanation) — Document explaining borrower anomalies (employment gaps, large deposits, credit inquiries). JARVIS auto-generates from templates with compliance citations.

SOC2 / PCI-DSS / GLBA — Security and privacy compliance frameworks. SOC2 (service organization controls), PCI-DSS (payment card security), GLBA (Gramm-Leach-Bliley financial privacy). CIPHER audits against all three.

Exponential Backoff — Retry strategy: delay = 2^attempt seconds. Used in task dispatch (max 3 retries), API calls, and watchdog restart. Prevents thundering herd on failures.

Token Bucket Rate Limiter — Controls request rate per domain. Tracks timestamps in sliding window (60s). Growth Ops browser client uses 30 req/min default with per-platform action budgets.
